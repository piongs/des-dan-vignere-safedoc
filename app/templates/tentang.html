🔐 Konsep Double Encryption

SecureDoc menerapkan dua lapis enkripsi secara berurutan (cascaded encryption). Lapis pertama menggunakan Vigenere Cipher — algoritma kriptografi klasik yang bekerja pada level karakter. Hasilnya kemudian dienkripsi lagi menggunakan DES-CBC — standar enkripsi blok modern. Dengan pendekatan ini, penyerang harus memecahkan dua lapisan sekaligus untuk mendapatkan plaintext.

① Vigenere Cipher
Algoritma kriptografi klasik · Bekerja pada level karakter · mod 26
📖 Sejarah & Konsep

Vigenere Cipher diciptakan oleh Blaise de Vigenère pada abad ke-16 (1553). Algoritma ini merupakan pengembangan dari Caesar Cipher — jika Caesar menggeser semua huruf dengan nilai tetap, Vigenere menggunakan kunci berulang sehingga setiap huruf digeser dengan nilai yang berbeda.

Algoritma ini bekerja pada level karakter alfabet (A–Z). Karakter non-alfabet seperti angka, spasi, tanda baca tidak diubah dan dilewati begitu saja. Kunci yang digunakan adalah serangkaian huruf yang diulang mengikuti panjang teks.

📐 Formula Matematika
// ENKRIPSI
Ci = (Pi + Ki) mod 26
// DEKRIPSI
Pi = (Ci - Ki + 26) mod 26
Pi = posisi huruf plaintext ke-i (A=0, B=1, ..., Z=25)
Ki = posisi huruf kunci ke-i (A=0, B=1, ..., Z=25)
Ci = posisi huruf ciphertext ke-i
mod 26 = sisa bagi 26 (jumlah huruf alfabet)

✅ Kelebihan Vigenere
  • Lebih kuat dari Caesar Cipher karena kunci berubah
  • Frekuensi huruf lebih sulit dianalisis
  • Sederhana dan mudah dipahami
  • Kunci fleksibel, bisa panjang atau pendek
⚠ Kelemahan Vigenere
  • Rentan terhadap analisis Kasiski jika kunci pendek
  • Hanya mengenkripsi huruf alfabet (A–Z)
  • Tidak aman jika digunakan sendiri untuk data modern
  • Dikombinasikan dengan DES untuk menutupi kelemahan ini

② DES-CBC (Data Encryption Standard)
Enkripsi blok simetris · Kunci 56-bit · 16 putaran Feistel · Mode CBC
📖 Sejarah & Konsep

DES dikembangkan oleh IBM dan ditetapkan sebagai standar enkripsi AS oleh NIST pada tahun 1977. DES adalah algoritma enkripsi blok simetris — artinya kunci yang sama digunakan untuk enkripsi dan dekripsi.

DES bekerja dalam blok 64-bit (8 byte) dengan kunci efektif 56-bit. Proses enkripsi melibatkan 16 putaran jaringan Feistel yang melakukan substitusi dan permutasi berulang. Mode CBC (Cipher Block Chaining) membuat setiap blok bergantung pada blok sebelumnya, sehingga pola yang sama dalam plaintext menghasilkan ciphertext yang berbeda.

📐 Detail Teknis DES-CBC
// ENKRIPSI (tiap blok)
Ci = DES_Enc(Pi XOR Ci-1, K_des)
// DEKRIPSI (tiap blok)
Pi = DES_Dec(Ci, K_des) XOR Ci-1
// FORMAT FILE .enc
[IV 8 byte] + [Ciphertext]
IV = Initialization Vector 64-bit (acak tiap enkripsi)
C0 = IV (untuk blok pertama)
K_des = kunci 8 byte dari SHA-256(kunci Anda)[:8]
Padding = PKCS#7 ke kelipatan 8 byte
Blok = 64-bit, 16 putaran Feistel per blok

✅ Kelebihan DES-CBC
  • Enkripsi pada level byte — mengenkripsi semua karakter
  • IV acak membuat enkripsi yang sama menghasilkan output berbeda
  • Mode CBC menyebarkan pola, lebih aman dari ECB
  • Standar yang sudah terbukti dan banyak dipelajari
⚠ Kelemahan DES
  • Kunci 56-bit sudah dianggap tidak aman untuk aplikasi modern
  • Rentan terhadap brute force dengan komputasi modern
  • Digantikan AES di industri, tapi masih relevan untuk pembelajaran
  • Dikombinasikan Vigenere untuk menambah kompleksitas

Cara kerja konkret SecureDoc dalam kode Python
Vigenere — Python
# Enkripsi Vigenere (mod 26) def vigenere_encrypt(plaintext, key): key = key.upper() result = [] ki = 0 for ch in plaintext: if ch.isalpha(): shift = ord(key[ki % len(key)]) - ord('A') base = ord('A') if ch.isupper() else ord('a') result.append(chr((ord(ch) - base + shift) % 26 + base)) ki += 1 else: result.append(ch) # non-alpha tidak diubah return ''.join(result)
DES-CBC — Python
# Enkripsi DES-CBC def des_encrypt(data: bytes, key_str: str): key = hashlib.sha256( key_str.encode()).digest()[:8] cipher = DES.new(key, DES.MODE_CBC) ct = cipher.encrypt( pad(data, DES.block_size)) # simpan IV di depan ciphertext return cipher.iv + ct # Dekripsi DES-CBC def des_decrypt(data: bytes, key_str: str): key = hashlib.sha256( key_str.encode()).digest()[:8] iv, ct = data[:8], data[8:] cipher = DES.new(key, DES.MODE_CBC, iv=iv) return unpad(cipher.decrypt(ct), DES.block_size)
📋 Alur Lengkap dalam Kode
Enkripsi: (1) Baca file → decode UTF-8 atau base64
(2) Jalankan vigenere_encrypt(teks, kunci_vigenere)
(3) Encode hasil ke bytes UTF-8
(4) Jalankan des_encrypt(bytes, kunci_des)
(5) Encode ke Base64 → simpan sebagai file .enc
Dekripsi: (1) Baca file .enc → decode Base64
(2) Jalankan des_decrypt(data, kunci_des) → ambil IV 8 byte pertama
(3) Decode bytes ke teks UTF-8
(4) Jalankan vigenere_decrypt(teks, kunci_vigenere)
(5) Hasil = plaintext asli

💡 Mengapa Mengkombinasikan Keduanya?

Vigenere sendiri lemah terhadap analisis frekuensi. DES sendiri sudah dianggap usang karena kunci 56-bit. Namun ketika dikombinasikan, penyerang harus memecahkan lapisan DES terlebih dahulu (mendapat output Vigenere), kemudian memecahkan Vigenere tanpa mengetahui panjang kuncinya. Dua algoritma yang masing-masing lemah, bila disusun berlapis, menghasilkan keamanan yang jauh lebih tinggi karena kompleksitas serangan meningkat secara eksponensial.
